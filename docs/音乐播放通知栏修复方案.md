# 音乐播放通知栏修复方案

## 问题描述

### 问题 1：音乐缓存初始化失败
启动时出现缓存目录初始化错误：
```
❌ [MusicCacheService] 缓存目录不可写: PathNotFoundException: Cannot delete file, path = '/data/user/0/com.example.star_bank/files/music_cache/.test' (OS Error: No such file or directory, errno = 2)
❌ [MusicCacheService] 初始化失败: Exception: 缓存目录不可写
```

### 问题 2：通知栏不显示
AudioService 配置正确，但播放音乐时通知栏不显示。

## 根本原因分析

### 问题 1 原因
测试文件写入后立即删除时，在某些 Android ROM 上文件可能已被系统清理，导致删除操作抛出 `PathNotFoundException`，进而导致整个缓存服务初始化失败。

### 问题 2 原因
1. 通知栏需要在特定时机主动触发状态广播
2. 初始化时没有正确设置 MediaItem 和播放状态
3. AudioService 配置参数冲突（`androidNotificationOngoing` 与 `androidStopForegroundOnPause` 不兼容）

## 解决方案

### 问题 1：优化缓存初始化逻辑

**文件**：`lib/services/music_cache_service.dart`

**关键修改**：
1. 简化测试文件命名（使用固定名称 `.write_test` 而不是时间戳）
2. 使用 `catchError` 捕获删除异常，不影响可写判断
3. 降级处理：目录不可写时自动禁用缓存功能，而不是抛出异常

```dart
// 验证目录是否可写 - 简化逻辑，避免文件删除问题
try {
  final testFile = File('${_cacheDir!.path}/.write_test');
  await testFile.writeAsString('test', flush: true);

  // 验证文件是否真的写入成功
  if (await testFile.exists()) {
    await testFile.delete().catchError((e) {
      // 删除失败不影响判断，只要能写入就说明目录可用
      debugPrint('⚠️ [MusicCacheService] 测试文件删除失败（不影响功能）: $e');
    });
  }

  debugPrint('✅ [MusicCacheService] 缓存目录可写');
} catch (e) {
  debugPrint('❌ [MusicCacheService] 缓存目录不可写: $e');
  // 降级处理：不抛出异常，而是禁用缓存功能
  cacheEnabled.value = false;
  debugPrint('⚠️ [MusicCacheService] 已自动禁用缓存功能');
}
```

### 问题 2：增强通知栏显示逻辑

**文件**：`lib/services/audio_handler.dart`

#### 修改 1：修复 AudioService 配置冲突

```dart
Future<AudioHandler> initAudioService() async {
  return await AudioService.init(
    builder: () => MusicHandler(),
    config: const AudioServiceConfig(
      androidNotificationChannelId: 'com.starbank.app.channel.audio.v6', // 新渠道ID
      androidNotificationChannelName: 'StarBank 音乐播放',
      androidNotificationChannelDescription: '音乐播放控制',
      androidNotificationOngoing: false, // 修复：与 androidStopForegroundOnPause=false 冲突
      androidStopForegroundOnPause: false, // 暂停时保持前台服务
      androidNotificationClickStartsActivity: true,
      androidShowNotificationBadge: true,
      notificationColor: Color(0xFFFFB27D),
      androidNotificationIcon: 'drawable/ic_stat_music_note',
    ),
  );
}
```

**重要**：`androidNotificationOngoing: true` 与 `androidStopForegroundOnPause: false` 不能同时使用，否则会导致编译错误。

#### 修改 2：优化初始化，确保通知栏立即显示

```dart
Future<void> _init() async {
  // 配置 AudioSession
  final session = await AudioSession.instance;
  await session.configure(const AudioSessionConfiguration.music());

  // 监听播放事件
  _player.playbackEventStream.listen(_broadcastState);
  _player.playerStateStream.listen((state) {
    _broadcastState(_player.playbackEvent);
  });

  // 设置初始 MediaItem（更友好的提示）
  mediaItem.add(const MediaItem(
    id: '__INIT__',
    title: 'StarBank 音乐',
    artist: '准备就绪',
  ));

  // 初始状态广播
  playbackState.add(PlaybackState(
    controls: [MediaControl.play],
    processingState: AudioProcessingState.idle,
    playing: false,
    systemActions: const {
      MediaAction.seek,
      MediaAction.seekForward,
      MediaAction.seekBackward,
    },
    androidCompactActionIndices: const [0],
    queueIndex: _queueIndex,
  ));

  // 延迟后再次广播，确保系统注册完成
  await Future.delayed(const Duration(milliseconds: 100));
  playbackState.add(/* 同上 */);
}
```

#### 修改 3：播放时立即显示加载状态

```dart
Future<Duration?> setAudioSourceAndPlay(
  AudioSource source,
  MediaItem item,
) async {
  await ready;

  // 1. 更新 MediaItem
  mediaItem.add(item);

  // 2. 立即广播 loading 状态，确保通知栏出现
  playbackState.add(PlaybackState(
    controls: [
      MediaControl.skipToPrevious,
      MediaControl.pause,
      MediaControl.skipToNext,
    ],
    systemActions: const {
      MediaAction.seek,
      MediaAction.seekForward,
      MediaAction.seekBackward,
    },
    androidCompactActionIndices: const [0, 1, 2],
    processingState: AudioProcessingState.loading,
    playing: false,
    updatePosition: Duration.zero,
    bufferedPosition: Duration.zero,
    speed: 1.0,
    queueIndex: _queueIndex,
  ));

  // 3. 设置音源并播放
  final duration = await _player.setAudioSource(source);
  await play();

  return duration;
}
```

#### 修改 4：播放时强制广播状态

```dart
@override
Future<void> play() async {
  await ready;
  debugPrint('🎵 [AudioHandler] play() called - starting playback');
  await _player.play();
  // 强制广播一次状态，确保通知栏更新
  _broadcastState(_player.playbackEvent);
}
```

#### 修改 5：增加调试日志

在关键方法中添加日志，方便排查问题：
- `_broadcastState()` - 记录状态广播
- `updateMediaItem()` - 记录 MediaItem 更新
- `play()` - 记录播放触发

## 测试验证

### 缓存服务测试
启动应用后检查日志：
- ✅ 正常：`[MusicCacheService] 缓存目录可写`
- ⚠️ 降级：`[MusicCacheService] 已自动禁用缓存功能`（不会导致崩溃）

### 通知栏测试
1. 启动应用，通知栏可能显示初始状态（可选）
2. 播放任意歌曲，通知栏应立即显示：
   - 歌曲标题和艺术家
   - 三个控制按钮：上一首 | 播放/暂停 | 下一首
   - 白色音符图标
3. 测试按钮功能：播放/暂停、切歌
4. 锁屏后检查锁屏界面的媒体控制

### 如果通知栏仍不显示

1. **检查权限**：系统设置 → 应用 → StarBank → 通知权限
2. **清除缓存**：卸载应用重新安装（清除旧通知渠道 v5）
3. **查看日志**：关注 `AudioHandler` 和 `AudioService` 相关日志

## 技术要点

1. **通知渠道版本管理**：每次修改通知配置时更新渠道 ID（v5 → v6），避免旧配置缓存
2. **前台服务保活**：`androidStopForegroundOnPause: false` 确保暂停时不被系统杀死
3. **状态广播时机**：在 MediaItem 更新后、音源设置前后都要广播状态
4. **降级处理**：非关键功能失败时降级而不是崩溃，提升应用稳定性
5. **调试友好**：关键流程添加清晰的日志，包含 emoji 标识便于快速定位
