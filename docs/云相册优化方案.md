# 云相册优化方案

## 修复内容

### 1. 优化图片选择器上传状态显示 ✅

**问题描述**：
- 在媒体选择器中，需要逐个点击图片才能验证是否已上传
- 用户体验差，无法快速识别哪些图片已经上传过

**解决方案**：
在打开媒体选择器时，自动批量检测首屏及附近图片的上传状态。

**文件**：`lib/pages/kin/baby_cloud_media_picker_page.dart`

**关键修改**：

```dart
Future<void> _loadMore() async {
  if (_path == null || _loadingMore || !_hasMore) return;
  _loadingMore = true;
  final next = await _path!.getAssetListPaged(page: _page, size: _pageSize);
  _page++;
  _assets.addAll(next);
  _assets.sort((a, b) => b.createDateTime.compareTo(a.createDateTime));
  _hasMore = next.length == _pageSize;
  _loadingMore = false;
  _safeSetState(() {});

  // 优化：加载第一页时预检测更多图片，后续页面只检测可见部分
  final warmupCount = _page == 1 ? 40 : 18;  // 首页检测40张，其他页18张
  final visibleWarmups = next.take(warmupCount).toList();
  unawaited(
    Future<void>.delayed(const Duration(milliseconds: 200))  // 缩短延迟
        .then((_) => _warmUploadedMarks(visibleWarmups)),
  );
}
```

**优化效果**：
- 首次打开选择器时，自动检测前 40 张图片的上传状态
- 后续滚动加载时，检测每页前 18 张图片
- 延迟从 350ms 降低到 200ms，响应更快
- 已上传的图片会自动显示标记，无需点击

### 2. 支持图片双指缩放查看 ✅

**问题描述**：
- 动态中的图片点击查看原图后，无法通过双指捏合放大
- `InteractiveViewer` 的手势被外层 `PageView` 拦截

**解决方案**：
动态检测缩放状态，在缩放时禁用 `PageView` 的滑动，恢复正常时重新启用。

**文件**：`lib/pages/kin/baby_cloud_media_detail_page.dart`

**关键修改**：

```dart
class _BabyCloudMediaDetailPageState extends State<BabyCloudMediaDetailPage> {
  late PageController _controller;
  late int _index;
  bool _isZoomed = false; // 跟踪是否处于缩放状态

  @override
  Widget build(BuildContext context) {
    final item = widget.items[_index];
    return Scaffold(
      body: Column(
        children: [
          Expanded(
            child: PageView.builder(
              controller: _controller,
              itemCount: widget.items.length,
              physics: _isZoomed
                  ? const NeverScrollableScrollPhysics() // 缩放时禁用滑动
                  : const PageScrollPhysics(),
              onPageChanged: (index) => setState(() => _index = index),
              itemBuilder: (_, index) {
                final current = widget.items[index];
                if (current.isAudio) return _AudioPreview(item: current);
                if (current.isVideo) return _VideoPreview(item: current);
                return Center(
                  child: InteractiveViewer(
                    minScale: 0.5,    // 最小缩放 50%
                    maxScale: 4.0,    // 最大放大 4 倍
                    onInteractionStart: (details) {
                      // 检测多指操作，禁用 PageView
                      if (details.pointerCount >= 2) {
                        setState(() => _isZoomed = true);
                      }
                    },
                    onInteractionUpdate: (details) {
                      // 持续检测缩放状态
                      final scale = details.scale;
                      final shouldDisable = scale != 1.0 || details.pointerCount >= 2;
                      if (shouldDisable != _isZoomed) {
                        setState(() => _isZoomed = shouldDisable);
                      }
                    },
                    onInteractionEnd: (details) {
                      // 交互结束后延迟重新启用 PageView（防止误触）
                      Future.delayed(const Duration(milliseconds: 300), () {
                        if (mounted && details.velocity.pixelsPerSecond.distance < 50) {
                          setState(() => _isZoomed = false);
                        }
                      });
                    },
                    child: BabyCloudMediaThumbnail(
                      item: current,
                      fit: BoxFit.contain,
                      backgroundColor: Colors.black,
                      preferOriginal: true,
                      showVideoBadge: false,
                    ),
                  ),
                );
              },
            ),
          ),
          // ... 底部信息栏
        ],
      ),
    );
  }
}
```

**功能特性**：
- ✅ 支持双指捏合缩放（0.5x ~ 4x）
- ✅ 缩放时自动禁用左右滑动，防止误触切换图片
- ✅ 缩放完成后自动恢复滑动功能
- ✅ 300ms 防抖延迟，避免快速手势误判
- ✅ 不影响原有的左右滑动切换图片功能

## 技术要点

### 1. 哈希验证优化策略

**现有机制**：
- 通过 SHA256 哈希值判断图片是否已上传
- `_remoteHashes` 存储所有已上传图片的哈希值
- `_hashCache` 缓存已计算的图片哈希

**优化策略**：
- **首屏优先**：首次加载时检测前 40 张（约 2-3 屏）
- **按需检测**：滚动加载时每页检测 18 张（约半屏）
- **异步非阻塞**：使用 `unawaited` 异步执行，不阻塞 UI
- **缓存复用**：已计算的哈希值会被缓存，避免重复计算

### 2. 手势冲突处理

**问题根源**：
`InteractiveViewer` 在 `PageView` 内部时，两者的手势识别会冲突：
- 单指水平滑动：`PageView` 切换图片
- 双指捏合：`InteractiveViewer` 缩放
- 单指拖动（已缩放）：`InteractiveViewer` 平移

**解决方案**：
1. **动态物理特性**：根据缩放状态切换 `PageView` 的 `physics`
2. **多指检测**：`pointerCount >= 2` 时禁用滑动
3. **缩放检测**：`scale != 1.0` 时禁用滑动
4. **防抖延迟**：交互结束后延迟 300ms 恢复，避免误触

### 3. 性能考量

**图片哈希计算**：
- 使用 `crypto` 包的 SHA256 算法
- 仅在必要时计算（首屏+可见区域）
- 计算结果缓存，避免重复

**状态更新**：
- 使用 `_safeSetState()` 保护状态更新
- 检查 `mounted` 和 `_closing` 标志
- 异步操作完成后才更新 UI

## 测试验证

### 图片选择器测试

1. **打开选择器**：
   - 观察首屏图片，已上传的图片应立即显示标记
   - 无需点击即可看到上传状态

2. **滚动加载**：
   - 向下滚动加载更多图片
   - 新加载的图片会自动检测并显示上传状态

3. **选择上传**：
   - 选择未上传的图片
   - 点击"上传"按钮
   - 已上传的图片无法被选中

### 图片缩放测试

1. **查看原图**：
   - 在动态详情页点击任意图片
   - 进入全屏查看模式

2. **缩放操作**：
   - 双指捏合放大：图片应正常放大（最大 4 倍）
   - 双指捏合缩小：图片应正常缩小（最小 0.5 倍）
   - 单指拖动（已缩放）：图片应平移，而不是切换到下一张

3. **切换图片**：
   - 双击或缩放到 1:1 后，左右滑动应正常切换图片
   - 不应与缩放手势冲突

4. **边界情况**：
   - 快速缩放后立即滑动：应有 300ms 延迟保护
   - 单张图片：缩放功能正常，无滑动需求

## 后续优化建议

1. **更智能的预检测**：
   - 使用 `VisibilityDetector` 检测可见图片
   - 仅检测视口内及附近的图片

2. **缓存持久化**：
   - 将哈希缓存持久化到本地存储
   - 下次打开时直接使用缓存

3. **批量哈希计算**：
   - 使用 `compute()` 在独立 isolate 中计算哈希
   - 避免阻塞主线程

4. **手势优化**：
   - 添加双击缩放功能
   - 支持旋转手势（可选）
